# XGBoost（勾配ブースティング）

## このノートブックでやること

「前の木の間違いを次の木が修正する」勾配ブースティングの高速実装 XGBoost で、タイタニックの生存予測を行う。

## アルゴリズム解説

### 勾配ブースティング = 間違いを修正し続ける

ランダムフォレストは「独立した木の多数決」だった。
勾配ブースティングは、**前の木が間違えた部分に注目して、次の木がそれを修正**するアプローチだ。

```
[ランダムフォレスト]
  木1(独立), 木2(独立), 木3(独立) ... → 多数決

[勾配ブースティング]
  木1 → 残差(誤差)を計算
       → 木2 が残差を学習 → 新たな残差を計算
                           → 木3 が残差を学習 → ...

最終予測 = 木1の予測 + 木2の補正 + 木3の補正 + ...
```

比喩で言えば：
- ランダムフォレスト = **100人が独立に採点して平均**をとる
- 勾配ブースティング = **1人目が採点 → 2人目が「ここ間違ってるよ」と修正 → 3人目がさらに修正...**

### ゼロつく1の勾配降下法との関連

ゼロつく1で学んだ勾配降下法を思い出してほしい：

```
重み = 重み - 学習率 × 勾配
```

勾配ブースティングでも同じ発想を使う。ただし、重みを更新する代わりに **新しい木を追加** する：

```
予測 = 前の予測 + 学習率 × 新しい木(残差を予測)
```

「勾配の方向に少しずつ改善していく」という本質は同じだ！

### XGBoost の位置づけ

XGBoost（eXtreme Gradient Boosting）は、勾配ブースティングの **高速・高精度な実装** だ。
内部で並列化や正則化などの最適化が施されており、Kaggle等のコンペで圧倒的な実績がある。

## ハイパーパラメータ

| パラメータ | 値 | 意味 |
|---|---|---|
| `n_estimators` | 500 | 木の本数（ブースティングの反復回数） |
| `learning_rate` | 0.03 | 学習率（各木の寄与の大きさ。小さいほど慎重） |
| `max_depth` | 3 | 木の最大深さ（浅い木で少しずつ修正） |
| `subsample` | 0.8 | 各木で使うデータの割合（過学習防止） |
| `colsample_bytree` | 0.8 | 各木で使う特徴量の割合（多様性を確保） |
| `reg_lambda` | 1.0 | L2正則化（重みを小さく保つ。ゼロつく1の Weight Decay と同じ） |

### 学習率（learning_rate）の役割

```
学習率が大きい(0.3): 各木の影響が大きい → 少ない木で収束するが過学習リスク
学習率が小さい(0.03): 各木の影響が小さい → 多くの木が必要だが汎化性能が高い

今回の設定: learning_rate=0.03 × n_estimators=500
→ ゆっくり、たくさん修正するスタイル
```

## ポイント・学び

- **勾配ブースティングは「間違いの修正」の連鎖**。ゼロつく1の勾配降下法と同じ「勾配の方向に改善」の発想
- **AUC=0.8561** はランダムフォレスト(0.8603)をわずかに下回る。チューニング次第で逆転する可能性がある
- **Accuracy の標準偏差=0.0291** は全モデル中最小。fold間のブレが少なく安定した予測ができている
- **learning_rate と n_estimators はトレードオフ**。学習率を下げるなら木の本数を増やす必要がある
- **正則化パラメータ（reg_lambda, max_depth）が過学習を防ぐ**。445件という少量データでは特に重要
