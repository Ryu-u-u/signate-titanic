# ニューラルネットワーク（MLPClassifier）

## このノートブックでやること

ゼロつく1で学んだニューラルネットワークそのもの（多層パーセプトロン）で、タイタニックの生存予測を行う。

## アルゴリズム解説

### ゼロつく1で学んだ NN そのもの

このモデルは、ゼロつく1で実装したニューラルネットワークと **本質的に同じもの** だ。

```
[ゼロつく1 で作った NN]
  入力(784) → Affine → ReLU → Affine → ReLU → Affine → Softmax → 10クラス分類

[今回の MLPClassifier]
  入力(14) → Affine → ReLU → Affine → ReLU → Affine → Sigmoid → 2クラス分類
```

違いは：
- **入力**: 画像ピクセル(784) → タイタニック特徴量(14)
- **出力**: 10クラス(数字0-9) → 2クラス(生存/死亡)
- **実装**: NumPyで手書き → scikit-learn の MLPClassifier が自動で逆伝播・最適化

### アーキテクチャ図

```
入力層(14)    隠れ層1(64)    隠れ層2(32)    出力層(1)
  ●─────────→  ●─────────→  ●─────────→  ●
  ●            ●            ●            生存確率
  ●            ●            ●
  ●            ●            ●
  ...          ...          ...
  ● (14個)     ● (64個)     ● (32個)

  各層間: Affine(重み付き和) → ReLU(活性化関数)
  出力層: Sigmoid → 確率 [0, 1]
```

パラメータ数: 14×64 + 64 + 64×32 + 32 + 32×1 + 1 = **3,073個**

### StandardScaler が必須な理由

ゼロつく1でも、MNIST画像を 0〜255 → 0.0〜1.0 に正規化してから入力した。
同じ理由で、ニューラルネットの入力は **スケールを揃える必要がある**。

```
正規化なし: age(0-80) と sex(0-1) のスケールが違いすぎる
            → 勾配の大きさが特徴量ごとにバラバラ
            → 学習が不安定になる

StandardScaler: 全特徴量を平均0、標準偏差1に変換
            → 勾配が安定し、学習がスムーズに進む
```

## ハイパーパラメータ

| パラメータ | 値 | 意味 |
|---|---|---|
| `hidden_layer_sizes` | [64, 32] | 隠れ層のニューロン数 |
| `activation` | relu | 活性化関数（ゼロつく1で学んだ ReLU） |
| `alpha` | 0.001 | L2正則化の強さ（Weight Decay） |
| `learning_rate_init` | 0.001 | 初期学習率（ゼロつく1の lr と同じ） |
| `early_stopping` | True | 検証スコアが改善しなくなったら学習を打ち切る |
| `max_iter` | 500 | 最大エポック数 |

## ノートブックの流れ

1. **Setup**: ライブラリ読み込み、設定ファイルの読み込み
2. **Data Loading**: v1特徴量パイプラインで14特徴量を生成
3. **Feature Engineering**: 使用する14特徴量の確認
4. **Cross-Validation**: 5-fold StratifiedKFold で評価
5. **Train Full Model**: 全学習データで再学習、テストデータを予測
6. **Submission**: 提出ファイル生成
7. **Analysis**: 学習曲線（Loss over iterations）を可視化

## 実験結果

> **交差検証（CV）って何？** → [01_preprocess/README.md](../01_preprocess/README.md) で詳しく解説している。
> データを5分割し、検証部分を入れ替えながら5回評価→平均する手法だ。
> 標準偏差が大きいほど「fold によってスコアがブレる＝不安定なモデル」という意味になる。

### 5-fold CV 結果

| Fold | Accuracy | F1 | LogLoss | AUC |
|---|---|---|---|---|
| 0 | 0.8202 | 0.7576 | 0.4563 | 0.8632 |
| 1 | 0.7640 | 0.6667 | 0.5006 | 0.9012 |
| 2 | 0.7416 | 0.6102 | 0.5935 | 0.7495 |
| 3 | 0.8090 | 0.7385 | 0.4686 | 0.8446 |
| 4 | 0.6629 | 0.3478 | 0.6618 | 0.5794 |
| **平均** | **0.7596** | **0.6241** | **0.5362** | **0.7876** |

標準偏差（Accuracy）: **0.0629**（全モデル中最大）

### なぜ性能が低いのか？

**AUC=0.7876** は全モデル中最低。主な原因は：

1. **データが少なすぎる**。3,073個のパラメータに対して445件のデータは圧倒的に少ない。ゼロつく1のMNISTは60,000件あった
2. **学習が不安定**。Fold 4 の AUC=0.5794（ほぼランダム）に見られるように、初期値やデータの分割によって大きくブレる
3. **表形式データでは NN の強みが活きにくい**。画像のような「空間構造」がないデータでは、決定木系のモデルが有利

### ゼロつく1 の NN との違い

| 項目 | ゼロつく1 | 今回 |
|---|---|---|
| 実装 | NumPy で手書き | scikit-learn が自動化 |
| 逆伝播 | 手動で計算グラフを書いた | `.fit()` で自動実行 |
| 最適化 | SGD を自分で実装 | Adam が内部で動作 |
| データ量 | 60,000件 | 445件 |

## ポイント・学び

- **MLPClassifier はゼロつく1で学んだ NN そのもの**。順伝播・逆伝播・勾配降下法を scikit-learn が自動で実行する
- **NN は大量データで輝く**。445件では過学習しやすく、不安定になる
- **標準偏差 0.0629 は「信頼できない」サイン**。fold ごとに結果が大きく変わるモデルは、本番環境でも不安定な予測をする
- **表形式データでは決定木系が強い**。NN が圧倒的に強いのは画像・テキスト・音声など、構造を持つデータだ
- **ゼロつく1の知識は無駄ではない**。勾配降下法・逆伝播・活性化関数の理解は、他の全アルゴリズムの理解にも役立つ
