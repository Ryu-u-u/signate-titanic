# 探索的データ分析（EDA）

## このノートブックでやること

タイタニック号の乗客データを「見る・数える・比べる」ことで、生存に影響する要因を探る。
モデルを作る前の"偵察"フェーズだ。

## EDA とは何か

**EDA（Exploratory Data Analysis）= 探索的データ分析** は、データをグラフや集計で眺めて「何が効きそうか」「どこに問題があるか」を発見する工程だ。

ゼロつく1ではMNISTの画像を表示して「数字の形を確認した」場面があったはず。
あれもEDAの一種だ。画像なら目で見ればわかるが、表形式データでは **分布・相関・欠損** を数値とグラフで確認する必要がある。

```
データの全体像を掴む
    ↓
特徴量ごとの分布を見る（単変量分析）
    ↓
特徴量と目的変数の関係を見る（二変量分析）
    ↓
特徴量同士の組み合わせを探る（多変量分析）
    ↓
発見をもとに特徴量エンジニアリングの方針を立てる
```

## データの全体像

```
学習データ: 445行 × 8列
評価データ: 446行 × 7列（survived なし）
ターゲット: survived（0=死亡, 1=生存）
クラス比率: 死亡 266人(59.8%) : 生存 179人(40.2%) ≒ 1.49:1
```

| カラム | 型 | 欠損(train) | 説明 |
|---|---|---|---|
| survived | int | 0 | 生存(1) or 死亡(0) ← 目的変数 |
| pclass | int | 0 | 客室クラス（1=上級, 2=中級, 3=下級） |
| sex | str | 0 | 性別（male/female） |
| age | float | 85 (19.1%) | 年齢 |
| sibsp | int | 0 | 同乗の兄弟/配偶者数 |
| parch | int | 0 | 同乗の親/子数 |
| fare | float | 0 | 運賃 |
| embarked | str | 2 (0.4%) | 乗船港（S/C/Q） |

## 主要な発見

### 1. 性別が最強の予測因子

```
女性の生存率: 77.6%
男性の生存率: 20.1%
→ 相関係数: 0.559（最も高い）
```

「女性と子供を先に」という救命ボートの方針が数字にはっきり出ている。

### 2. 客室クラスの影響が大きい

```
1等客室: 68.5% 生存
2等客室: 44.3% 生存
3等客室: 25.8% 生存
→ 相関係数: -0.358
```

### 3. pclass × sex の交互作用が最も強力

```
1等女性: 生存率が非常に高い
3等男性: 生存率が非常に低い
→ この組み合わせが最も予測力を持つ
```

### 4. 運賃は客室クラスと強く相関

fare の分布は右に大きく偏っている（skewness=3.78）。
`log1p(fare)` で対数変換すると skewness=0.48 まで改善する。

### 5. 年齢の欠損は「ランダムではない」（MAR）

```
年齢欠損率:
  1等客室: 10.2%
  2等客室:  4.1%
  3等客室: 29.2%
```

3等客室の乗客ほど年齢が記録されていない。
→ 全体の中央値で補完するのではなく、**客室クラス別の中央値**で補完すべきだ。

### 6. 家族サイズにU字型の関係

- 一人旅（family_size=1）: 生存率が低い
- 2〜4人の家族: 生存率が高い
- 5人以上の大家族: 生存率が再び低下

## ポイント・学び

- **EDAはモデリングの土台**。データを見ずにモデルを回すのは、地図なしで山に登るのと同じだ
- **欠損値は「なぜ欠けているか」まで考える**。age の欠損がpclassに依存していることを発見したから、適切な補完方法を選べる
- **相関 ≠ 因果**。fareとpclassの相関が高いのは「高い客室クラスは運賃が高い」という当然の関係。多重共線性に注意が必要
- **EDAの発見は仮説**。最終的には交差検証（CV）で検証する必要がある
