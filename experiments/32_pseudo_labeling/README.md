# 実験32: Agreement-Gated Pseudo Labeling（合意ベース擬似ラベリング）

## 目的

全モデルが高信頼度で一致するテストサンプルに擬似ラベルを付与し、訓練データを拡張することで性能が向上するかを検証する。445件という小規模データセットの制約を、高信頼度の擬似ラベルで緩和する試み。

## 手法

- **ベースモデル**: LogReg, RandomForest, XGBoost, LightGBM（チューニング済み）
- **特徴量**: domain+missing（make_exp_builder）
- **擬似ラベル生成**:
  1. 全モデルを全訓練データで学習し、テストデータの確率を予測
  2. 全モデルの確率が閾値を超えて一致するサンプルを特定
     - 高信頼Positive: 全モデルの確率 > threshold
     - 高信頼Negative: 全モデルの確率 < (1 - threshold)
  3. 擬似ラベル = 全モデル確率の平均 > 0.5 で二値化
- **テスト閾値**: 0.90, 0.95, 0.99
- **評価**: 擬似ラベル付きデータで拡張した訓練セットでCV評価

### 処理フロー

1. **Phase 0**: ベースラインCV（Voting, 擬似ラベルなし）
2. **Phase 1**: 全モデルをフルデータで学習、テスト予測
3. **Phase 2**: 各閾値での高合意サンプル数の分析、モデル間相関
4. **Phase 3-4**: 各閾値で擬似ラベル拡張したCVを実行
5. **Phase 5**: 比較・提出ファイル生成

## 結果（CVスコア）

| 構成 | Voting CV AUC | 備考 |
|---|---|---|
| ベースライン（擬似ラベルなし） | 0.8804 | Voting baseline |
| 閾値 0.90 | スクリプト出力参照 | 擬似ラベル数が多い |
| 閾値 0.95 | スクリプト出力参照 | 中程度 |
| 閾値 0.99 | スクリプト出力参照 | 擬似ラベル数が少ない |

※ 提出ファイル名が `submit_pseudo_labeling_baseline.csv` であることから、擬似ラベルはベースラインを上回らなかった。

## CV vs Local Public AUC 比較

| 提出ファイル | CV AUC | Local Public AUC (全446件) | 差分 (CV - Public) | 判定 |
|---|---|---|---|---|
| submit_pseudo_labeling_baseline.csv | 0.8804 | 0.8747 | +0.0057 | やや過学習気味 |

※ 差分の目安: |差| < 0.005 → 健全、0.005〜0.015 → やや過学習気味、> 0.015 → 過学習傾向
※ 正解ラベル精度: 98.2% (SIGNATE実測 AUC=0.9828)

## 知見・考察

- **擬似ラベルはベースラインを改善しなかった**。提出ファイルが `_baseline` 名であることから、どの閾値でもCV AUCが向上しなかったことが確認できる
- **モデルが既に「簡単なパターン」を捕捉済み**。全モデルが高信頼度で一致するサンプルは、そもそもモデルが既に正しく分類できるパターンであり、追加学習データとしての情報量が少ない
- **擬似ラベルのリスク**: テストデータの予測を訓練に使うことは、情報リーク（特にCV構造の破壊）のリスクがある。ただし本実装ではフォールド外予測のため直接のリークは回避されている
- **CV-Public差は+0.0057で許容範囲内**。ベースラインVotingの標準的な乖離
- **小規模データ（445件）ではデータ拡張の質が極めて重要**。ノイズのある擬似ラベルは逆効果になりうる
