## 今後の改善方針

### 特徴量エンジニアリング
- チケット番号やキャビン番号からの情報抽出（元データにあれば）
- 外部データ（乗客名簿等）との結合
- 特徴量選択（不要な特徴量の削除で過学習を防ぐ）

### モデルチューニング
- LightGBM のハイパーパラメータ調整（チューニング余地が大きい）
- XGBoost の early_stopping の活用
- Optuna 等による自動ハイパーパラメータ探索

### アンサンブルの改善
- 重み付き Voting（モデルごとに異なる重みを設定）
- Stacking のメタ学習器を Ridge に変更
- Blending（ホールドアウトベース）の試行

### バリデーション戦略
- Repeated StratifiedKFold（繰り返しCVで分散を低減）
- NestedCV（ハイパーパラメータ探索のリーク防止）
