# アンサンブル学習（Ensemble）

## このノートブックでやること

これまで作った複数のモデルを「合体」させて、単体モデルを超える予測精度を目指す。

## アルゴリズム解説

### アンサンブル = 複数モデルの「合体」

個々のモデルにはそれぞれ得意・不得意がある：

```
ロジスティック回帰: 線形な関係に強い、非線形に弱い
ランダムフォレスト: 非線形に強い、外挿に弱い
XGBoost/LightGBM: 残差を修正する力が強い
ニューラルネット: 柔軟だが小データで不安定
```

これらを組み合わせると、**1つのモデルの弱点を他のモデルが補う**。
ちょうど、チームメンバーそれぞれの強みを活かして協力するのと同じだ。

### なぜアンサンブルが強いのか

```
[モデルA] 患者Xの予測: 0.7 (生存)  ← 少し高すぎ
[モデルB] 患者Xの予測: 0.5 (五分五分) ← 少し低すぎ
[モデルC] 患者Xの予測: 0.6 (やや生存) ← まあまあ

平均: (0.7 + 0.5 + 0.6) / 3 = 0.6 ← 一番正確に近い！
```

各モデルの誤差が **異なる方向にバラつく** なら、平均をとることで誤差が相殺される。
これが「多様性 → 誤りの相殺」の原理だ。

### Soft Voting（予測確率の平均）

最もシンプルなアンサンブル手法。各モデルの **予測確率を平均** する。

```
                  LogReg  RF    XGB   LGBM   → 平均
乗客A(生存確率):   0.85   0.78  0.82  0.80   → 0.8125
乗客B(生存確率):   0.20   0.30  0.25  0.22   → 0.2425
```

- 実装が簡単で、追加の学習が不要
- 全モデルを対等に扱う（重み付けなし）

### Stacking（メタ学習器が最終判断）

より高度なアンサンブル手法。各モデルの予測を **新しい特徴量** として、**メタ学習器** が最終予測を行う。

```
[Stage 1: ベースモデルの予測]
  LogReg → 予測確率 p1
  RF     → 予測確率 p2
  XGB    → 予測確率 p3
  LGBM   → 予測確率 p4

[Stage 2: メタ学習器]
  入力: [p1, p2, p3, p4]
  メタ学習器(LogisticRegression) → 最終予測

→ メタ学習器が「どのモデルをどれだけ信頼するか」を自動で学習
```

注意: Stage 1 の予測は **CVの out-of-fold 予測** を使う（学習データ自身の予測を使うとリークになる）。

## ノートブックの流れ

1. **Setup**: ライブラリ読み込み、全モデルの定義
2. **Data Loading**: v1特徴量パイプラインで14特徴量を生成
3. **Individual Model CV**: 単体モデル（LogReg, RF, XGB, LGBM）のCV結果を再計算
4. **Soft Voting**: VotingClassifier(voting="soft") でアンサンブル
5. **Stacking**: StackingClassifier でアンサンブル
6. **Comparison Table**: 全モデルのAUCを一覧表示・棒グラフで比較
7. **Train Best & Submit**: ベストモデル（Voting）で提出ファイル生成

## 実験結果

> **交差検証（CV）って何？** → [01_preprocess/README.md](../01_preprocess/README.md) で詳しく解説している。
> すべてのモデルを同じ5-fold分割で評価しているので、公平な比較ができる。

### 全モデル比較

| モデル | Accuracy | F1 | LogLoss | AUC |
|---|---|---|---|---|
| **Voting** | **0.8000** | **0.7426** | **0.4413** | **0.8639** |
| Stacking | 0.8022 | 0.7448 | 0.4446 | 0.8636 |
| RF | 0.8045 | 0.7510 | 0.4557 | 0.8603 |
| XGB | 0.8000 | 0.7389 | 0.4686 | 0.8561 |
| LogReg | 0.8000 | 0.7410 | 0.4566 | 0.8536 |
| LGBM | 0.7865 | 0.7290 | 0.5383 | 0.8429 |

**Soft Voting（AUC=0.8639）** が最高スコアを記録！

### Voting vs 単体モデルの差

```
Voting AUC:      0.8639
最良単体(RF) AUC: 0.8603
→ 改善: +0.0036

小さな差に見えるが、4つのモデルの予測を平均しただけで
追加コストなしに改善している。これがアンサンブルの力だ。
```

### Voting vs Stacking

```
Voting:   AUC=0.8639
Stacking: AUC=0.8636
→ ほぼ同等（差: 0.0003）
```

445件の小さなデータでは、Stacking の「メタ学習器の学習」自体が過学習しやすい。
シンプルな Voting で十分な性能が出ている。

## ポイント・学び

- **アンサンブルは「合体」の力**。異なるモデルの多様な視点を組み合わせることで、単体モデルを超えられる
- **Soft Voting は最もシンプルで効果的**。実装が簡単で、今回のベストスコアを記録した
- **Stacking はデータが多いときに真価を発揮**。445件では Voting との差が出にくい
- **アンサンブルの前提は「多様性」**。似たようなモデルを組み合わせても効果は薄い。LogReg(線形) + RF(バギング) + XGB(ブースティング) のように、異なるアプローチを混ぜるのがコツ
- **LogLoss=0.4413 は全モデル中最良**。予測確率の信頼度（calibration）もアンサンブルで改善されている
