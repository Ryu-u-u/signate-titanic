# ランダムフォレスト（Random Forest）

## このノートブックでやること

多数の決定木を組み合わせた「多数決」モデルで、タイタニックの生存予測を行う。

## アルゴリズム解説

### 決定木の直感

決定木は、**Yes/No の質問を繰り返してデータを分類**するモデルだ。

```
      性別は女性？
      /        \
    Yes         No
    /             \
  客室は1等?     → 死亡 (多数)
  /      \
Yes       No
 ↓         ↓
生存      客室は2等?
          /      \
        Yes       No
         ↓         ↓
        ...       死亡
```

人間が「この人は生き残れたか？」を推理するときの思考プロセスと同じだ。
「まず性別を確認、次に客室クラスを見て...」と条件分岐で絞り込んでいく。

ゼロつく1のニューラルネットが「数値の重み付け」で判断するのに対し、決定木は **「条件分岐」** で判断する。まったく異なるアプローチだ。

### ランダムフォレスト = 多数の決定木の多数決

1本の決定木は、データの些細な違いに敏感で不安定（過学習しやすい）。
そこで、**たくさんの決定木を作って多数決**をとる。これがランダムフォレストだ。

```
[バギング（Bootstrap Aggregating）の仕組み]

元データ (445件)
    ↓ ランダムに復元抽出（445件、重複あり）
    決定木1 → 予測: 0.7
    ↓ ランダムに復元抽出（445件、重複あり）
    決定木2 → 予測: 0.3
    ↓ ランダムに復元抽出（445件、重複あり）
    決定木3 → 予測: 0.8
    ...
    ↓ ランダムに復元抽出
    決定木500 → 予測: 0.6

最終予測 = 全決定木の予測確率の平均
         = (0.7 + 0.3 + 0.8 + ... + 0.6) / 500
```

さらに、各分岐で使える特徴量も **ランダムに制限** する（`max_features=sqrt`）。
→ 木どうしが「違う視点」から判断するので、多様性が生まれ、多数決の精度が上がる。

### なぜ多数決が強いのか

直感的に言えば：

> 1人の専門家の意見より、100人の「そこそこ詳しい人」の多数決のほうが正確

これが **「集合知」** の原理であり、ランダムフォレストの強さの源泉だ。

## ハイパーパラメータ

| パラメータ | 値 | 意味 |
|---|---|---|
| `n_estimators` | 500 | 決定木の本数（多いほど安定、計算は増える） |
| `max_depth` | 5 | 木の最大深さ（深いほど複雑な分岐が可能） |
| `min_samples_leaf` | 3 | 葉ノードの最小サンプル数（小さいと過学習） |
| `max_features` | sqrt | 各分岐で使える特徴量数（√14 ≒ 3.7 → 4個） |

## ノートブックの流れ

1. **Setup**: ライブラリ読み込み、設定ファイルの読み込み
2. **Data Loading**: v1特徴量パイプラインで14特徴量を生成
3. **Feature Engineering**: 使用する14特徴量の確認
4. **Cross-Validation**: 5-fold StratifiedKFold で評価
5. **Train Full Model**: 全学習データで再学習、テストデータを予測
6. **Submission**: 提出ファイル生成
7. **Analysis**: 特徴量重要度のバーチャートを可視化

### 特徴量重要度

ランダムフォレストは各特徴量が予測にどれだけ貢献したかを「重要度」として出力する。
sex, fare, age, pclass が上位に来る傾向がある。

## ポイント・学び

- **決定木は直感的に理解しやすい**。「Yes/No の質問の木」として可視化できるので、予測の理由が説明しやすい
- **ランダムフォレストは決定木の弱点（不安定さ）を多数決で克服**する。バギングと特徴量のランダム選択がポイント
- **StandardScaler は不要**。決定木は条件分岐（x < 閾値）で判断するので、スケールの影響を受けない
- **AUC=0.8603** はロジスティック回帰(0.8536)を上回っている。非線形な関係（年齢のU字型効果など）を自然に捉えられる強みだ
- **特徴量重要度は「参考」程度**。相関の高い特徴量があると重要度が分散するので、過信は禁物
